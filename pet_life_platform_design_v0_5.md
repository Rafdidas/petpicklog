# 반려동물 생활 정보 플랫폼 설계서 v0.5

## 1. 문서 목적

이 문서는 기존 `반려동물 생활 플랫폼 설계서 v0.4`를 기반으로, 서비스 방향을 한 단계 더 구체화한 5차 설계서이다.

기존 v0.4에서는 반려동물 용품 가격 비교 서비스에서 출발해, 동물병원 정보와 반려생활 가이드를 포함하는 `반려동물 생활 플랫폼`으로 방향을 확장했다.

v0.5에서는 여기에 공공데이터 기반 장소·업소 정보를 본격적으로 포함한다.

핵심은 다음과 같다.

```txt
공공데이터 기반 반려생활 장소 정보
+
네이버 쇼핑 API 기반 반려동물 용품 가격 확인
```

즉, 이 서비스는 단순 가격 비교 서비스가 아니라 반려동물과 함께 생활하는 데 필요한 정보를 한곳에서 확인하는 생활형 정보 플랫폼을 목표로 한다.

---

## 2. 서비스 방향

### 2.1 기존 방향

```txt
반려동물 용품 가격 비교·정보 서비스
```

### 2.2 v0.4 방향

```txt
반려동물 생활 플랫폼
```

### 2.3 v0.5 방향

```txt
공공데이터 기반 반려생활 정보 + 반려동물 용품 가격 확인 플랫폼
```

v0.5에서는 서비스의 중심축을 아래 두 가지로 나눈다.

```txt
1. 공공데이터 기반 반려생활 장소 정보
2. 네이버 쇼핑 API 기반 반려동물 용품 가격 확인
```

공공데이터는 사용자가 실제 생활에서 찾는 장소와 업소 정보를 제공한다.

네이버 쇼핑 API는 사용자가 자주 구매하는 사료, 간식, 배변용품, 위생용품 등의 가격을 확인하는 역할을 한다.

## 3. 서비스 한 줄 정의

```txt
반려동물과 함께하는 생활에 필요한 장소, 여행, 업소, 용품 가격 정보를 한곳에서 확인하는 반려생활 정보 플랫폼
```

조금 더 짧게 표현하면 다음과 같다.

```txt
공공데이터와 쇼핑 API로 확인하는 반려생활 정보 플랫폼
```

## 4. 서비스 핵심 가치

### 4.1 사용자 관점 핵심 가치

```txt
반려동물과 갈 수 있는 장소를 찾는다.
우리 동네 동물병원, 동물약국, 미용, 위탁관리 업소를 확인한다.
자주 사는 반려동물 용품의 가격을 확인한다.
관심상품을 저장하고 가격 변화를 다시 확인한다.
공공데이터 출처와 최신성 안내를 통해 정보를 보수적으로 제공한다.
```

### 4.2 서비스 관점 핵심 가치

```txt
가격 비교만으로는 약한 차별성을 공공데이터 기반 정보 서비스로 보완한다.
반려동물 생활과 관련된 공공데이터를 한곳에 모아 탐색성을 높인다.
네이버 쇼핑 API는 구매 전 가격 확인 기능으로 명확하게 역할을 제한한다.
기존 구현 기능을 버리지 않고 IA와 포지셔닝을 확장한다.
```

## 5. 포지셔닝

이 서비스는 다나와처럼 상품 가격을 확인하는 흐름을 일부 참고하지만, 전체 정체성은 가격 비교 사이트가 아니다.

서비스 정체성은 다음에 가깝다.

```txt
반려동물 생활 정보 허브
```

비슷한 서비스와 비교하면 다음과 같다.

```txt
다나와식 가격 확인
+
멍냥보감식 반려생활 정보
+
공공데이터 기반 지역 정보
```

다만 MVP에서는 커뮤니티, AI 추천, 지도 SDK, 복잡한 리뷰 기능은 제외한다.

초기에는 아래 흐름에 집중한다.

```txt
장소를 찾는다.
정보를 확인한다.
지도 또는 외부 링크로 이동한다.
용품 가격을 검색한다.
관심상품으로 저장한다.
가격 변화를 확인한다.
```

## 6.2 네이버 쇼핑 API

네이버 쇼핑 API는 아래 용도로 사용한다.

```txt
반려동물 용품 검색
상품 후보 표시
현재 확인 가격 표시
관심상품 저장
저장 당시 가격과 현재 확인 가격 비교
구매처 이동
```

네이버 쇼핑 API는 가격 확인의 기준으로 사용하되, `최저가 보장`처럼 오해될 수 있는 문구는 사용하지 않는다.


## 10.3 메뉴명 변경 이유

### 장소 찾기

기존 `동물병원`은 범위가 좁다.

v0.5에서는 동물병원뿐 아니라 동물약국, 미용, 위탁관리, 동반여행 정보까지 포함하므로 `장소 찾기`가 더 적합하다.

### 가격 확인

`용품 검색`보다 사용자의 목적에 가깝다.

사용자는 상품 자체를 검색하기보다 구매 전 가격을 확인하려는 목적이 크다.

### 관심상품

`저장한 상품`보다 쇼핑 서비스 문맥에 자연스럽다.

## 12.5 raw_data를 남기는 이유

공공데이터 API는 데이터마다 필드명이 다를 수 있다.

초기부터 모든 필드를 완벽하게 정규화하면 개발 부담이 커진다.

따라서 공통 필드는 정규화하고, 원본 데이터는 `raw_data`에 JSON 형태로 보관한다.

```txt
공통 검색과 카드 표시에 필요한 필드만 정규화한다.
카테고리별 세부 정보는 raw_data에서 꺼내 표시한다.
추후 자주 쓰는 필드는 별도 컬럼으로 승격한다.
```

```txt
4. public_pet_places upsert

```

## 14. 데이터 동기화 전략

## 14.1 초기 방식

초기에는 실시간 호출보다 DB 저장 후 조회하는 방식을 권장한다.

```txt
공공데이터 API → 서버 동기화 → Supabase 저장 → 서비스 조회
```

이유는 다음과 같다.

```txt
공공데이터 API 응답 속도와 장애 영향을 줄일 수 있다.
검색과 필터를 서비스 DB 기준으로 안정적으로 처리할 수 있다.
외부 API 구조가 바뀌어도 프론트 영향을 줄일 수 있다.
```

## 17. 홈과 쇼핑 기능의 연결

공공데이터와 쇼핑 API가 따로 놀지 않게 연결한다.

예시는 다음과 같다.

```txt
동반여행 카테고리 → 이동장, 배변봉투, 휴대용 물병 추천 검색어
동물병원 카테고리 → 넥카라, 이동장, 위생패드 추천 검색어
동물약국 카테고리 → 영양제, 구강관리, 피부관리 추천 검색어
동물미용 카테고리 → 샴푸, 브러시, 발톱깎이 추천 검색어
위탁관리 카테고리 → 사료, 간식, 배변패드 추천 검색어
```

이 연결을 통해 서비스는 단순 데이터 모음이 아니라 반려생활 흐름을 제안하는 플랫폼이 된다.

## 24. 최종 개발 판단

v0.5 방향은 기존 기능을 버리지 않고 서비스의 차별성을 강화하는 방향이다.

네이버 쇼핑 API만 사용하는 경우 서비스가 단순 상품 검색 보조 도구처럼 보일 수 있다.

하지만 공공데이터 기반 장소·업소 정보를 함께 제공하면, 서비스는 다음 포지션을 가질 수 있다.

```txt
반려동물 생활 정보 플랫폼
```

따라서 최종 방향은 다음과 같이 확정한다.

```txt
공공데이터 기반 반려생활 장소 정보
+
네이버 쇼핑 API 기반 반려동물 용품 가격 확인
+
관심상품 가격 변화 관리
```

이 구조는 현재 구현된 `상품 검색`, `관심상품`, `가격 기록`, `동물병원 조회` 기능을 유지하면서도, 서비스 정체성을 더 넓고 명확하게 만든다.
